ichosynth wiring — TŒRN port on hand-soldered hardware
======================================================

Hand-wired, point-to-point, NO custom PCB.
Teensy 4.1 (with BOTH 16 MB PSRAM chips soldered — mandatory) running the real
TŒRN firmware (by SP_/soundpauli, https://toern.live).

Pins live in the build sources, NOT in a config.h:
  - teensy/libraries/.../i2cEncoderLibV2.h   -> ICHOS_ENC_PINS  (encoders)
  - teensy/libraries/.../FastTouch.h         -> ICHOS_BTN_PINS  (buttons)
  - teensy/build_toern.py                    -> SWITCH_1/2/3 remap

These pins deliberately avoid TŒRN's hard-coded GPIO and the Teensy Audio
Shield's reserved pins. Do NOT use pins 2/3/4 (TŒRN internal use) or 24 (was an
optional LED strip, removed in this build).


> LED-Matrix (WS2812 16x16, chainable to 32x16):
DIN    - Teensy: 17
+5V    - Teensy: +5V
GND    - Teensy: GND


> Encoders (KY-040, turn + push), {CLK, DT, SW}, left -> right.
  Each encoder also: + -> 3V3, GND -> GND.

Encoder E1 (left):
CLK    - Teensy: 5
DT     - Teensy: 22
SW     - Teensy: 15

Encoder E2:
CLK    - Teensy: 32
DT     - Teensy: 33
SW     - Teensy: 41

Encoder E3:
CLK    - Teensy: 9
DT     - Teensy: 14
SW     - Teensy: 16

Encoder E4 (right):
CLK    - Teensy: 37
DT     - Teensy: 38
SW     - Teensy: 39


> Buttons — 3 tact switches (PLAY / MENU / REC).
  Each: one leg -> Teensy pin, other leg -> GND. Active-low (internal pull-up).
B1 PLAY - Teensy: 25
B2 MENU - Teensy: 26
B3 REC  - Teensy: 28


> Audio (Teensy Audio Adaptor, SGTL5000). Easiest: stack on the Teensy.
  Hand-wired signals:
MCLK   - Teensy: 23
BCLK   - Teensy: 21
LRCLK  - Teensy: 20   (WS)
TX     - Teensy: 7    (DIN to codec)
RX     - Teensy: 8    (DOUT from codec)
SDA    - Teensy: 18   (I2C, shared with OLED)
SCL    - Teensy: 19   (I2C, shared with OLED)
+3     - 3V3
GND    - GND
  (SD is used from the Teensy 4.1's built-in slot, not the audio board slot.)


> Audio I/O jacks — 3x 6.35mm (1/4") MONO TS, wired to the Audio Shield's
  LINE IN / LINE OUT pads (NOT to Teensy GPIO). The on-board 3.5mm headphone
  jack is kept for monitoring. Every 6.35mm: tip = signal, sleeve = GND.

  Line IN  (mono, 1 jack):  tip -> Audio Shield LINE IN  L   ; sleeve -> GND
  Line OUT (stereo, 2 jacks):
     jack L:                tip -> Audio Shield LINE OUT L   ; sleeve -> GND
     jack R:                tip -> Audio Shield LINE OUT R   ; sleeve -> GND
  Headphone:                on-board 3.5mm stereo jack (no wiring needed)

  Notes:
   - Line In is mono: TŒRN records/monitors the LEFT line-in as a mono sample.
     LINE IN is already the firmware's default record input (no firmware change).
   - Line Out is left stereo (two jacks L+R); the codec DAC drives Line Out and
     the headphone jack together. Optional 10uF DC-blocking caps in series only
     if connected gear hums; direct wiring is normally fine.


> OLED (SSD1306 0.96" 128x64 I2C). Shares the audio I2C bus.
  Replaces TŒRN's encoder RGB-ring feedback (shows channel/mode/transport/BPM/
  volume/page). Default I2C address 0x3C.
SDA    - Teensy: 18
SCL    - Teensy: 19
VCC    - 3V3
GND    - GND

Firmware / flashing:
------------------------------------------------
Build:  python teensy/build_toern.py   ->  teensy/firmware/toern.hex
        (needs arduino-cli + teensy:avr core; builds at -O1 / opt=o1std)
Flash:  python teensy/build_toern.py --flash    (CLI, teensy_loader_cli)
        or use the one-click GUI flasher in _FLASHER for a prebuilt .hex.
See teensy/README.md for details. Board: Teensy 4.1 (USB type handled by the
build script).
